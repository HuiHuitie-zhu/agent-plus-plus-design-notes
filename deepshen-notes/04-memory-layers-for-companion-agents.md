# 长期记忆要分层，不要混成一锅粥

> 长期陪伴型 AI 最怕的不是忘记，而是把所有东西都当成同一种记忆。

## 一句话

长期记忆不应该只有一个“memory”。原始日志、日记摘要、用户画像、事实缓存、情绪状态必须分层存放、分层读取、分层过期。

否则记忆系统很快会从“更了解用户”变成“污染上下文”。

## 为什么“都记住”不是答案

很多长期 AI 系统会从一个朴素愿望出发：

```text
用户说过的都记住。
```

听起来很好，实际很危险。

因为不同内容的性质完全不同：

- 原始对话是证据。
- 日记是解释。
- 用户画像是提炼。
- 天气搜索是短期事实。
- 情绪状态是系统内部变量。
- 模型曾经说错的话是风险样本，不一定该回灌。

如果它们都被塞进同一个上下文池，模型会分不清什么是事实、什么是推测、什么已经过期、什么只是一次错误输出。

## 一个更稳的分层方式

最小可用分层可以是：

```text
1. 原始日志层
   append-only，记录真实发生过的对话。

2. 日记 / 摘要层
   对原始日志的阶段性整理，可失败，可重写，但不能替代原文。

3. 用户画像层
   长期偏好、稳定事实、关系理解。写入要谨慎，最好带来源日期。

4. 感知缓存层
   天气、搜索、日历等短期事实。必须有来源和过期时间。

5. 情绪状态层
   系统内部状态，不直接等同于用户记忆。
```

这些层的读写规则不同，不能混用。

## 原始日志必须比智能整理更可靠

一条底线：

```text
原始日志先落盘，智能整理后发生。
```

模型可以写日记，可以做摘要，可以提炼画像，但这些都是衍生物。衍生物失败时，不能影响原始记录。

所以：

- 日记失败，不写空日记。
- 摘要失败，不删除原文。
- 画像失败，不覆盖旧画像。
- 压缩失败，继续使用未压缩窗口。

长期陪伴系统里，丢失经历比少一点智能更糟。

## 用户手写区和模型提炼区要隔离

如果允许模型更新用户画像，必须有硬边界。

一个简单结构：

```markdown
<!-- MANUAL_START -->
## 手写区
用户明确写下的事实和偏好。
<!-- MANUAL_END -->

<!-- AUTO_START -->
## 自动提炼区
模型根据日志提炼的观察，必须带来源日期。
<!-- AUTO_END -->
```

规则：

- 手写区只读，不被模型覆盖。
- 自动区可以重写，但要保留来源。
- 自动区内容不能悄悄升级成铁律。

这能防止“模型猜测”污染“用户明确表达”。

## 不要把模型的怪回复喂回去当风格样本

原型里一个很有代表性的坑是：模型偶尔说怪话后，如果下一轮把它自己的旧回复原文放回上下文，它会把怪话当成风格示范继续模仿。

这类回灌很危险。

更稳的做法是：

- 最近历史主要注入用户最近说过的话。
- AI 自己的旧回复只作为必要上下文，不作为风格样本。
- 已确认的怪回复要能被标记、排除或降权。

长期系统的记忆不仅要记住好东西，也要知道哪些东西不能强化。

## 记忆要有来源和生命周期

每条衍生记忆最好至少有：

```text
source        来自哪一天 / 哪条日志
created_at    何时生成
confidence    置信度
expires_at    是否会过期
```

不是所有记忆都该永久有效。

“用户喜欢某种表达风格”可能稳定。

“用户今天很累”可能明天就过期。

“今天会下雨”两小时后就可能没意义。

没有生命周期的记忆，最后会变成上下文里的旧报纸。

## 可迁移的原则

- 原始日志是证据，摘要和画像是衍生物。
- 衍生物失败不能影响原始记录。
- 用户手写内容高于模型推断。
- 短期事实必须带来源和过期时间。
- 不要让模型旧错误进入正反馈循环。
- 记忆系统的目标不是“全都记住”，而是“该用时知道该信哪一层”。

## English Abstract

Long-term companion memory should be layered. Raw logs, summaries, user profiles, perception caches, and emotional state have different lifecycles and trust levels.

The goal is not to remember everything equally. The goal is to preserve evidence, separate derived interpretations, avoid memory pollution, and ensure the model knows which layer to trust.
